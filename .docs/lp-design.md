# PoohMa（プーマ）LPビジュアル構成・アセット・デザインベンチマーク統合定義書

## 1. デザインベンチマーク詳細分析

PoohMaのLPは、デベロッパーライクな「洗練された硬質感」と、一般家族向けの「圧倒的な安心感・使いやすさ」を両立させるため、以下の5つの優れたプロダクトからデザインDNAを抽出し、融合させます。

### ① Nani（Nani Translation）の特徴と分析

* **極限のミニマリズム:** 無駄な装飾やグラデーションを徹底的に排除。極細の境界線（`border-geist`: `#e4e4e7` 等）と、緻密に計算されたホワイトスペース（余白）のみで世界の美しさと知性を表現。
* **ノイズレスなUIレイヤー:** ユーザーの視線誘導を妨げる要素が一切なく、情報の優先順位が1ピクセル単位で整理されている。
* **信頼感への寄与:** 「一切の誤魔化しがない、誠実で高度なテクノロジープロダクトである」という硬質な信頼感をユーザーに与える。

### ② LINEの特徴と分析

* **究極に低い認知負荷:** 説明書やチュートリアルを一切読まなくても、3歳から90歳までが直感的に操作・理解できるアクセシビリティの極致。
* **会話の記号化（チャットバブル）:** 「緑と白の吹き出し」というUI自体が、現代人にとって「日常のコミュニケーション、家族の会話、親しみやすさ」を説明不要で想起させる強力な記号として機能する。
* **信頼感への寄与:** テック特有の「難しそう、冷たい、自分には関係ない」という心理的距離感を一瞬でゼロにする。

### ③ Notionの特徴と分析

* **「動き」による直感理解:** 静止画のスクリーンショットではなく、文字がタイピングされたりブロックが動いたりする「数秒のループ動画・GIF」をLPに多用し、文脈を読ませずに体験させる。
* **知的文房具のアイコニックな世界観:** モノトーン主体のシャープなUIに、あえて「ゆるいドット絵や線画イラスト」を組み合わせることで、先進性と人間味を高度にブレンドしている。
* **インタラクティブな情報整理:** 多機能さをダラダラ縦に並べず、「タブ切り替え」等を用いてLPの縦の長さをスッキリと抑え、認知過多を防ぐ。

### ④ マネーフォワード（Money Forward ME）の特徴と分析

* **不安を先回りして潰すロジック:** 資産や口座情報という究極のプライベートデータを扱う上で、ユーザーが最も恐怖するポイント（例：「勝手に出金されないか？」）を特定し、「閲覧データのみ取得、出金パスワードは預からない」とLP上で力強く明言。
* **印象のハイブリッド配色:** 堅牢さと金融機関並みの安全性を表す「深いネイビー/ブルー」をベースに、前向きでアクティブな「ブランドオレンジ」をアクセントに配し、堅苦しさを中和。
* **社会的証明のデカ文字配置:** 「利用者数No.1」「1500万人突破」などの客観的実績を巨大なフォントで配置し、集団心理による安心感を形成。

### ⑤ noteの特徴と分析

* **血の通ったミニマリズム:** 白とニュアンスグレーを基調とした広大な余白を使いつつも、無機質にならずに「体温や生活」を感じさせる絶妙なレイアウト。
* **ブランドカラーの「点」の運用:** 全体を緑色に染めるのではなく、ロゴや最重要CTAボタンにのみ「noteグリーン」をピンポイントで配置し、主役（コンテンツやユーザー）を徹底的に引き立てる。
* **手触り感のあるビジュアル:** 抽象的な3DやCGを避け、素朴なイラストや実際の人の温かみのあるアセットを配置することで、プラットフォーム全体の安全なコミュニティ感を演出。

## 2. ベンチマークを統合した全体のビジュアル構成案

5つのベンチマークから得た知見を統合した、LP全体のセクション設計です。

| セクション | レイアウト・配色・実装イメージ | 採用したベンチマークのDNAと視覚的意図 |
| :--- | :--- | :--- |
| **① ヒーロー**<br>(ファーストビュー) | ・背景：完全な白（`#ffffff`）<br>・左側にマネフォ流の大きな数字実績（「家族の忘れた！を1分で解決」等）と強いCTA。<br>・右側に**メインビジュアル（画像A）**。スマホ画面内はNotion流の動くデモ動画。 | **【マネフォ × Notion】**<br>クリーンな先進性を担保しつつ、動的な画面と強力なフックで「怪しくない、本物のサービス」である印象を1秒で与える。 |
| **② 課題提起**<br>(共感セクション) | ・背景：note流の広大でゆったりした余白（`py-24`以上）。<br>・中央に、家族のリアルなやり取りを再現した**LINE風チャットUI（画像B）**を配置。 | **【LINE × note】**<br>「あ、これ我が家のことだ」と一瞬で当事者意識を持たせる。文字を読ませるのではなく、記号として状況を理解させる。 |
| **③ コアバリュー**<br>(機能のタブ切り替え) | ・背景：白（`#ffffff`）<br>・Nani風の極細のグレーのボーダーで区切られたモダンなUI。<br>・縦に並べず、Notion流の**「タブ切り替え（安全 / 簡単 / 自由）」**を採用。<br>・各タブに**機能アイコン（画像E群）**。 | **【Nani × Notion】**<br>LPが縦にダラダラ長くなるのを防ぎ、ユーザーが知りたい価値（「なぜ安全なのか」「どうCSVで出せるのか」）をノイズレスに選択して読ませる。 |
| **④ 実際の画面**<br>(操作デモ) | ・背景：note風の薄いマスタードクリーム色（`#fffbeb`）で画面全体の冷たさを中和。<br>・中央に、3〜5秒の操作ループを見せる**スマホ動画モックアップ（画像C）**。 | **【Notion × note】**<br>パパやママが実際にスマホを操作し、実パスワードではなく「ヒント」がスッと登録・共有される様子を動きで見せ、「これならできる」の確信を作る。 |
| **⑤ 安全性の証明**<br>(先回りリスク否定) | ・背景：完全な白（`#ffffff`）。<br>・マネフォ流の「私たちは実パスワードを1文字も預かりません」という強烈な太字メッセージ。<br>・その下（または横）に、3ステップの横流れ**セキュリティ図解（画像D）**を大きく配置。 | **【マネフォ × note】**<br>ユーザーの「一番の恐怖（パスワード流出）」を先回りして完全に潰す。銀行レベルの暗号化というロジックを、3ステップの図解で優しく視覚的に翻訳。 |
| **⑥ FAQ / CTA** | ・背景：ごく薄いグレー（`#fafafa`）。<br>・アコーディオンUIで整頓。最後に大きなブランドオレンジのCTAボタン。 | **【Nani × マネフォ】**<br>noteのようにブランドカラーを「点」で目立たせ、最後の疑問を解消してスムーズにコンバージョンへ導く。 |

## 3. 必要な画像・動画アセットの詳細（アセットリスト）

### 画像A：メインビジュアル（ヒーロー用）

* **用途:** ヒーローセクション（ファーストビュー）の右側
* **種類:** 親しみやすい2Dフラット線画イラスト（Notionのイラストに温かみのある色を乗せたタッチ）
* **詳細:** パパ、ママ、子供、おじいちゃんが笑顔で1台のスマートフォンを覗き込み、「あ、これね！」と納得している様子。人物の背景には薄いオレンジの円や四角のオブジェクトを敷き、ファミリー向けの温かみを演出。
* **特殊仕様:** 左側に配置される大型のスマートフォン画面部分は、実際のアプリがカチャカチャ動くデモ動画（Notion流）をコーディングまたは動画編集ではめ込めるよう、**「完全なグリーンバック（クロマキー用カラー）」**でくり抜いたマスターアセットとする。

### 画像B：課題表現ビジュアル（チャットUI）

* **用途:** 課題提起セクション
* **種類:** クリーンなUIコンポーネント画像（またはCSS/SVGでの実実装）
* **詳細:** LINEのトーク画面を模した、シンプルで認知負荷の低いチャットバブル。
  * ママ：「お父さん、アマプラのパスワード変えた？ ログインできない」
  * パパ：「あれ？メモした紙どこやったっけ…」
  * おじいちゃん：「ルーターの裏のやつじゃダメなのか？」
  家族間の「パスワードどこだっけ問題」の生々しさを、洗練されたフォント（Geist SansやNoto Sans）で綺麗に整頓して視覚化する。

### 画像C：操作ループ動画モックアップ

* **用途:** 実際の画面セクション
* **種類:** 3〜5秒の短尺高解像度ループ動画（または高画質GIF）
* **詳細:** 実際のアプリ画面（またはLP用に要素を大きくしたデモ画面）の操作キャプチャ。パパが「Netflix」の項目をタップすると、パスワードそのものは非表示（●●●●）のまま、その下に【ヒント：ポチの誕生日＋いつもの4桁】と大きく滑らかに表示されるアニメーション。デバイスの背景に薄いドロップシャドウ（`shadow-sm`）を適用し、モダンに浮き上がらせる。

### 画像D：セキュリティ図解（インフォグラフィック全体）

* **用途:** 「安全性の証明セクション」のメイン図解
* **種類:** クリーンでロジカルな3ステップのインフォグラフィック（Figma等でSVGコンポーネントを組み合わせて構築する構成図）
* **詳細:** 「①手元でロック（ユーザーのスマホ内）」➔「②暗号化された箱で送信（インターネットの移動）」➔「③金庫に保管（PoohMaサーバー）」というデータの流れを、細い線画、ブランドオレンジ、そして信頼感を補強するディープネイビーで表現した横流れの図解。
* **図解の内部構成素材：「お手上げ管理人クマ」のイラスト:**
  * **配置場所:** 上記図解の【ステップ3：金庫に保管】のすぐ横にレイアウト。
  * **詳細:** 確定した「クマの鍵アイコン（`image_2.png`）」のクマの顔の造形・線の太さをそのまま流用し、帽子をかぶった「PoohMaの管理人クマ」にアレンジ。
  * **ポーズ・表情:** 「困ったな、僕たちも家族専用のカギ（暗号鍵）を持っていないから中身を開けられないや…」という、優しくコミカルにお手上げしている表情とポーズ（両手を軽く広げる、または少し困った眉にする）。
  * **素材仕様:** 図解のレイアウトやテキストの調整がFigma上で柔軟に行えるよう、装飾や文字を完全に排除した**「シンプルな一色（単色）背景」**の独立したキャラクターアセットとして作成し、図解にはめ込む。

### 画像E群：アイコンセット（正方形アセット）

* **用途:** ロゴ、コアバリューの機能説明、各種バッジ等
* **種類:** 角丸正方形のベースに、細すぎず太すぎない洗練された線画を施したアセット群。基本色はブランドオレンジ（`#f97316`）をベースとした温かみのある配色。
* **バリエーション（全6種）:**
  1. **ブランドロゴ本体:** 確定デザインである「優しいクリーム色のクマの顔の斜め下に、斜めに交差する鍵」が配置された、オレンジの角丸正方形アイコン。
  2. **文字入りブランドロゴ:** 上記のロゴ本体の下部に、洗練されたフォントで「PoohMa」のブランド文字が美しく配置されたバージョン。
  3. **機能アイコン（安全/家族）:** クマの頭上に家族のシルエットが入ったハートマークがあしらわれた、家族共有を示すアイコン。
  4. **機能アイコン（簡単/検索）:** クマの顔の前に虫眼鏡が配置された、ヒントを素早く見つけられることを示すアイコン。
  5. **機能アイコン（自由/エクスポート）:** 鍵の斜め下に「外向きの矢印がついた書類」が配置された、いつでもデータをCSV等で自由に取り出せることを示すアイコン。
  6. **機能アイコン（グローバル/ネットワーク）:** 鍵の斜め下に地球儀マークが配置された、Webベースでどこからでも安全にアクセスできることを示すアイコン。
